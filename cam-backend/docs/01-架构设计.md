# cam-backend 架构设计文档

## 项目概述

cam-backend 是一个基于 Go 的 USB 摄像头视频流管理后端，负责接收客户端通过 RTMP 推送的 H.264 视频流，通过 SRS 媒体服务器转为 HLS 落盘存储，并提供直播（HTTP-FLV）和回放（HLS m3u8）两种观看模式。

## 端口分配

| 端口 | 服务 | 协议 | 用途 |
|------|------|------|------|
| **1935** | SRS | RTMP | 接收客户端推流 |
| **1985** | SRS | HTTP | SRS 管理 API |
| **8080** | SRS | HTTP | HTTP-FLV 直播分发 + HLS 静态文件 |
| **9090** | cam-backend | HTTP | Go 服务（API + Webhook） |

## 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| 语言 | Go 1.25 | mise 管理 |
| HTTP 框架 | Gin | 高性能，社区成熟 |
| 数据库 | SQLite | 轻量，gorm 驱动 |
| ORM | GORM | 自动迁移，开发效率高 |
| 流媒体服务器 | SRS 6 | Docker 部署，host 网络模式 |
| 容器化 | Docker Compose | SRS 部署 |

## 数据模型

### Device（设备表）

以 MAC 地址为唯一标识，记录设备在线状态。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uint | 主键 |
| mac | string | MAC 地址，唯一索引 |
| name | string | 设备名称（默认=mac） |
| online | bool | 是否在线 |
| stream_key | string | 当前推流的 stream key |
| ip | string | 设备 IP |
| created_at | time | 创建时间 |
| updated_at | time | 更新时间 |

### StreamSession（推流会话）

记录每次推流的起止时间，用于统计和排查。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uint | 主键 |
| mac | string | 设备 MAC |
| stream_key | string | 流名称 |
| app | string | SRS 应用名 |
| client_id | string | SRS 客户端 ID |
| ip | string | 推流 IP |
| started_at | time | 推流开始时间 |
| stopped_at | time* | 推流结束时间（nullable） |

### RecordFile（录制文件）

记录 HLS 切片信息，支持按设备+日期查询回放。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uint | 主键 |
| mac | string | 设备 MAC |
| date | string | 日期 YYYY-MM-DD |
| file_path | string | 文件路径 |
| duration | float64 | 切片时长(秒) |
| created_at | time | 创建时间 |

## SRS 配置要点

- **HLS 切片**：`hls_fragment 10s`，`hls_window 60s`，显式声明
- **HLS 录制**：`hls_cleanup off` + `hls_dispose 0`，所有 ts 文件永久保留
- **HTTP-FLV**：`http_remux` 开启，前端可用 flv.js 播放
- **Webhook**：8 个回调全部注册到 Go 服务 `:9090`
- **网络模式**：Docker host 模式，不做端口映射
