# 前端与客户端接入指南

## 一、部署启动

### 1. 启动 SRS

```bash
cd cam-backend/deps/srs
docker compose up -d
```

验证 SRS 运行：

```bash
curl http://localhost:1985/api/v1/versions
```

### 2. 启动 Go 服务

```bash
cd cam-backend
go run .
# 或编译后运行
go build -o cam-backend . && ./cam-backend
```

验证服务运行：

```bash
curl http://localhost:9090/health
# {"status":"ok"}
```

## 二、客户端接入（推流端）

客户端指 USB 摄像头采集端，需要将 H.264 视频流通过 RTMP 推送到 SRS。

### 推流地址格式

```
rtmp://{server_ip}:1935/live/{stream_name}?token={token}&mac={mac_address}
```

| 参数 | 说明 | 示例 |
|------|------|------|
| server_ip | 服务器 IP | `192.168.1.1` |
| stream_name | 流名称（建议用设备标识） | `camera01` |
| token | 鉴权 token（当前未校验，可传任意值） | `test123` |
| mac | 设备 MAC 地址，**必传**，作为设备唯一标识 | `AA:BB:CC:DD:EE:FF` |

### 推流示例

#### 使用 FFmpeg 测试推流

```bash
# 从 USB 摄像头采集并推流
ffmpeg -f v4l2 -i /dev/video0 \
  -c:v libx264 -preset ultrafast -tune zerolatency \
  -f flv "rtmp://192.168.1.1:1935/live/camera01?token=test&mac=AA:BB:CC:DD:EE:FF"
```

```bash
# 用本地视频文件模拟推流（测试用）
ffmpeg -re -i test.mp4 \
  -c:v libx264 -f flv \
  "rtmp://localhost:1935/live/camera01?token=test&mac=AA:BB:CC:DD:EE:FF"
```

#### 使用 OBS 推流

1. 设置 → 流 → 服务选择「自定义」
2. 服务器：`rtmp://192.168.1.1:1935/live`
3. 串流密钥：`camera01?token=test&mac=AA:BB:CC:DD:EE:FF`

#### 嵌入式/单片机推流

推流端需实现 RTMP 客户端，推送 H.264 编码流。常见方案：

- **ESP32-S3**：使用 `esp-rtmp` 库
- **树莓派**：直接用 FFmpeg
- **其他 MCU**：使用轻量 RTMP 库如 `librtmp`

关键参数：
- 编码：H.264 Baseline Profile
- 分辨率：建议 640×480
- 帧率：建议 15-30fps
- 码率：建议 500kbps - 1Mbps

### 推流状态反馈

推流成功后，SRS 会触发 `on_publish` Webhook，Go 服务自动：
1. 在数据库中创建/更新设备记录
2. 标记设备为 online
3. 创建推流会话记录

断流后自动触发 `on_unpublish`，标记设备 offline。

## 三、前端接入（Web 播放端）

### 3.1 直播播放（HTTP-FLV + flv.js）

#### 步骤

1. 调用 API 获取直播地址
2. 用 flv.js 播放 FLV 流

#### 获取直播地址

```javascript
const mac = 'AA:BB:CC:DD:EE:FF';
const resp = await fetch(`http://${SERVER}:9090/api/v1/live/${mac}`);
const { code, data } = await resp.json();

if (code === 0) {
  // data.flv_url = "http://192.168.1.1:8080/live/camera01.flv"
  playLive(data.flv_url);
} else {
  console.log('设备离线');
}
```

#### flv.js 播放

```html
<script src="https://cdn.jsdelivr.net/npm/flv.js/dist/flv.min.js"></script>
<video id="player" controls autoplay muted></video>

<script>
function playLive(url) {
  if (!flvjs.isSupported()) {
    alert('浏览器不支持 flv.js');
    return;
  }

  const player = flvjs.createPlayer({
    type: 'flv',
    url: url,
    isLive: true,
  });

  player.attachMediaElement(document.getElementById('player'));
  player.load();
  player.play();
}
</script>
```

#### Vue 3 组件示例

```vue
<template>
  <div>
    <video ref="videoRef" controls autoplay muted style="width: 100%"></video>
    <p v-if="!online">设备离线</p>
  </div>
</template>

<script setup>
import { ref, onMounted, onBeforeUnmount } from 'vue';
import flvjs from 'flv.js';

const props = defineProps({ mac: String });
const videoRef = ref(null);
const online = ref(false);
let player = null;

onMounted(async () => {
  const resp = await fetch(`/api/v1/live/${props.mac}`);
  const { code, data } = await resp.json();

  if (code !== 0) {
    online.value = false;
    return;
  }

  online.value = true;

  player = flvjs.createPlayer({
    type: 'flv',
    url: data.flv_url,
    isLive: true,
  });

  player.attachMediaElement(videoRef.value);
  player.load();
  player.play();
});

onBeforeUnmount(() => {
  if (player) {
    player.pause();
    player.unload();
    player.detach();
    player.destroy();
    player = null;
  }
});
</script>
```

### 3.2 录像回放（HLS + hls.js）

#### 步骤

1. 获取设备有录像的日期列表
2. 获取某天的 m3u8 文件列表
3. 用 hls.js 或原生 `<video>` 播放

#### 获取录制日期

```javascript
const mac = 'AA:BB:CC:DD:EE:FF';
const resp = await fetch(`/api/v1/playback/${mac}/dates`);
const { data: dates } = await resp.json();
// dates = ["2026-03-19", "2026-03-18"]
```

#### 获取 m3u8 地址

```javascript
const date = '2026-03-19';
const resp = await fetch(`/api/v1/playback/${mac}/${date}/url`);
const { data } = await resp.json();
// data.files = [{ name: "camera01.m3u8", url: "/hls/AA:BB:CC:DD:EE:FF/2026-03-19/camera01.m3u8" }]
```

#### hls.js 播放

```html
<script src="https://cdn.jsdelivr.net/npm/hls.js/dist/hls.min.js"></script>
<video id="playback" controls></video>

<script>
function playHLS(url) {
  const video = document.getElementById('playback');

  // Safari 原生支持 HLS
  if (video.canPlayType('application/vnd.apple.mpegurl')) {
    video.src = url;
    video.play();
    return;
  }

  // 其他浏览器用 hls.js
  if (Hls.isSupported()) {
    const hls = new Hls();
    hls.loadSource(url);
    hls.attachMedia(video);
    hls.on(Hls.Events.MANIFEST_PARSED, () => video.play());
  }
}
</script>
```

// 获取所有设备

const sessions = await fetch(`/api/v1/devices/${mac}/sessions`).then(r => r.json());

## 四、完整流程总结

```
客户端推流                         前端播放
    │                                │
    │ RTMP + H.264                   │  1. GET /api/v1/devices
    │ ?token=xx&mac=xx               │     → 获取设备列表
    ▼                                │
┌────────┐                           │  2. GET /api/v1/live/{mac}
│  SRS   │ ─── Webhook ──→ Go服务    │     → 获取FLV直播地址
│  :1935 │                  :9090    │     → flv.js 播放
└────┬───┘                           │
     │                               │  3. GET /api/v1/playback/{mac}/dates
     ├─→ HTTP-FLV (:8080) ──────────┘     → 获取录像日期
     │   前端 flv.js 直接连接              GET /api/v1/playback/{mac}/{date}/url
     │                                     → 获取m3u8地址
     └─→ HLS 落盘 (data/hls/)             → hls.js 播放
         Go :9090/hls/ 静态服务    ────────┘
```

## 五、注意事项

1. **跨域**：如果前端和后端不同源，需要在 Gin 添加 CORS 中间件
2. **MAC 地址编码**：URL 路径中的 `:` 可能需要 URL 编码（`%3A`），或统一使用 `-` 分隔符
3. **FLV 直播延迟**：flv.js 延迟约 1-3 秒，属正常
4. **HLS 延迟**：HLS 直播延迟较大（20-30s），回放无影响
5. **鉴权**：当前所有接口未做鉴权，生产环境需实现 token 校验
6. **SRS HTTP 端口**：SRS 占用 `8080`，前端开发时 dev server 注意避开此端口
